<p className="lead">バイブコーディングは、AIと会話しながらコード、文章、資料などを作る進め方です。目的や判断までAIに任せきりにせず、出てきた結果を人間が確認しながら進めます。</p>

## 人間が担当すること

- 何を作るかを決める
- どこまで任せるかを決める
- 出てきた結果を確認する
- 次に直す点を伝える

AIは作業を進める相手です。目的、材料、確認基準がないまま任せると、見た目だけ整ったものが返ってくることがあります。

## AIに任せやすいこと

- たたき台を作る
- 長いメモを整理する
- エラーの原因候補を分ける
- 文章の読みにくい箇所を指摘する
- ページ構成やチェックリストを提案する

## 任せすぎない方がよいこと

- 他の人に渡したり、使ってもらったりしてよいかの最終判断
- 事実確認をせずに断定すること
- 個人情報や秘密情報の扱い
- 読者や利用者への責任がある判断

## まず覚える考え方

完璧なプロンプトを探すより、AIが作業しやすい状態を作ります。
作業場所、材料、目的、確認方法が見えていれば、指示は短くても通じやすくなります。
