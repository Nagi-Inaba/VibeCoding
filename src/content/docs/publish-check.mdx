---
title: 公開する前に
description: GitHub Pagesや公開URLを使う前に、見せてよいものだけが出る状態か確認する。
---

<p className="lead">公開は「動いたから押すボタン」ではありません。ほかの人が見られる場所へ出す前に、見せてよいものだけが入っているかを確認します。</p>

## まず公開の種類を確認する

GitHub Pagesは、リポジトリの中にある静的なページを公開URLとして見せる仕組みです。便利ですが、公開されたページや公開リポジトリに入っている内容は、ほかの人から見える可能性があります。アクセス制御された共有場所ではないので、URLが広まっても困らない内容だけを置きます。

初心者は、最初にこの3つだけ分けます。

<table className="prompt-matrix">
  <thead>
    <tr><th>状態</th><th>見ること</th><th>次の行動</th></tr>
  </thead>
  <tbody>
    <tr><td>練習</td><td>自分だけが試すページか</td><td>公開しない。手元やプレビューで確認する。</td></tr>
    <tr><td>共有</td><td>知っている人にだけ見せたいか</td><td>GitHub Pagesだけでは制限共有にしない。アクセス制御のある共有方法を使うか、URLが広まっても困らない内容だけにする。</td></tr>
    <tr><td>公開</td><td>誰が見ても困らないか</td><td>リンク、権利、個人情報、秘密情報、スマホ表示まで確認する。</td></tr>
  </tbody>
</table>

## 公開前に見るもの

公開前は、AIに「大丈夫そうです」と言わせるだけでは弱いです。実際のファイル、表示、リンク、公開URLを確認します。

<div className="check-grid">
  <div className="check-card">
    <h3>入っている内容</h3>
    <ul>
      <li>個人情報、APIキー、トークン、内部資料名が入っていないか</li>
      <li>公開してよい画像、名前、会社名だけになっているか</li>
      <li>元資料や下書きを、そのまま公開フォルダへ入れていないか</li>
      <li>AIが作った文章の事実確認をしたか</li>
    </ul>
  </div>
  <div className="check-card">
    <h3>実際の表示</h3>
    <ul>
      <li>公開URLを開いて、トップページが表示されるか</li>
      <li>主要リンクをクリックできるか</li>
      <li>スマホ幅で文字や表がはみ出していないか</li>
      <li>404ページや戻る導線が分かるか</li>
    </ul>
  </div>
  <div className="check-card">
    <h3>戻し方</h3>
    <ul>
      <li>直前に動いていた状態を保存したか</li>
      <li>公開を止める方法や、前の状態に戻す候補を確認したか</li>
      <li>公開後に直す人、相談する人、相談用メモの形を決めたか</li>
      <li>修正履歴や作業記録を残したか</li>
    </ul>
  </div>
</div>

## Webアプリなら追加で見る

ログイン、フォーム、保存機能、外部API、決済、個人データがあるものは、ただの公開ページより重く見ます。

<table className="prompt-matrix">
  <thead>
    <tr><th>見ること</th><th>確認する理由</th></tr>
  </thead>
  <tbody>
    <tr><td>APIキーを直接書いていないか</td><td>見える場所に置くと、第三者に使われる可能性がある。</td></tr>
    <tr><td>環境変数に分けているか</td><td>公開されるフロントエンドでは、環境変数の値もブラウザ側に見えることがある。秘密APIキーはサーバー側やAPIプロキシなど、外から見えない場所で扱う。</td></tr>
    <tr><td>ログインと権限を分けているか</td><td>ログインできることと、全部を見てよいことは別。</td></tr>
    <tr><td>ユーザーごとのデータが混ざらないか</td><td>他人の情報が見える事故を防ぐ。</td></tr>
    <tr><td>入力値を確認しているか</td><td>変な文字や想定外の値で壊れるのを防ぐ。</td></tr>
    <tr><td>エラー時に秘密情報が表示されないか</td><td>ログや画面に内部情報が出ることがある。</td></tr>
    <tr><td>依存パッケージを確認したか</td><td>古い依存や危険な依存をそのまま公開しない。</td></tr>
    <tr><td>削除、バックアップ、問い合わせ先があるか</td><td>公開後に問題が起きたとき戻りやすくする。</td></tr>
  </tbody>
</table>

分からない項目があるなら、人に使わせる前に止まります。練習用や自分用で試すことと、他人のデータを扱うことは分けて考えます。

## AIへの頼み方

公開前は、確認だけを先に頼みます。

<div className="copy-block">
<pre>{`公開する前の確認をしてください。
まだ修正や公開操作はしないでください。

確認してほしいこと:
- 公開してよいファイルだけが入っているか
- 個人情報、APIキー、トークン、内部資料名がないか
- 公開URLで主要ページとリンクが表示されるか
- スマホ幅で読めるか
- Webアプリの場合、ログイン、権限、入力値、エラー表示に危ない点がないか
- 問題があったときに戻す候補があるか

問題がある場所、理由、直し方を表で出してください。
確認が終わるまで、まだファイル編集や公開操作はしないでください。`}</pre>
</div>

公開してよいか迷うときは、公開ではなく相談用メモを作ります。

<div className="copy-block">
<pre>{`公開してよいか迷っています。
秘密情報は伏せたまま、別の人やAIに相談するためのメモを作ってください。

含めるもの:
- 何を公開したいか
- 誰が見る想定か
- まだ不安なこと
- 確認済みのこと
- 伏せた情報の種類
- 次に判断してほしいこと`}</pre>
</div>
