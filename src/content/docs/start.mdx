<p className="lead">最初からフォルダ構成を細かく作る必要はありません。まず作業フォルダを1つだけ作り、「ここで何をしたいか」をAIに伝えます。</p>

## 1. 作業フォルダを1つ作る

最初に人間がやるのは、作業場所を1つ決めることです。

```text
my-first-project/
```

この中に、最初から `sources/` や `outputs/` を手で作らなくても大丈夫です。  
必要なフォルダやファイルは、AIに作ってもらいます。

## 2. やりたいことを宣言する

作業フォルダを開いたら、最初の指示はこれをそのまま貼って使えます。`〇〇` だけ自分の作りたいものに置き換えます。

<div className="copy-block">
<pre>{`このフォルダを作業場所にします。
ここで、〇〇を作りたいです。

最終的には、□□まで広げたいです。
ただし今回は、まず△△だけ作りたいです。

まず、必要最低限のフォルダとファイルを作ってください。

最低限ほしい構成:
- README.md: このフォルダで何をするか、進め方、次に読むものを書く
- sources/: 元資料、参考リンク、入力データを置く
- notes/: 考え途中のメモ、方針、整理前の文章を置く
- outputs/: 完成物、提出物、公開するものを置く
- logs/: 作業記録、確認結果、次回やることを残す

Codexで作業する場合:
- AGENTS.md を作り、このフォルダでの作業ルール、成果物の置き場所、確認方法を書いてください。

Claudeで作業する場合:
- CLAUDE.md を作り、このフォルダでの作業ルール、成果物の置き場所、確認方法を書いてください。

使っていないツール向けの設定ファイルは作らなくて大丈夫です。
まず最小の形まで進めてください。
作業後に、作ったファイルとフォルダ、次に読むべきファイルを短く教えてください。`}</pre>
</div>

もっと短く始めたいときは、次の形でも大丈夫です。

<div className="copy-block">
<pre>{`このフォルダを作業場所にします。
ここで、〇〇を作りたいです。
最終的には□□まで広げたいですが、今回はまず△△だけ作りたいです。
README.md、sources/、notes/、outputs/、logs/ を最小構成として作ってください。
Codexを使う場合は AGENTS.md、Claudeを使う場合は CLAUDE.md も作ってください。
まず最小の形まで進めてください。`}</pre>
</div>

将来のゴールは、軽く伝えておいた方が安全です。最初の作り方が、あとから拡張しにくい形になるのを避けられます。

## 3. 細かい準備はAIに任せる

AIに任せてよいものは、たとえば次のようなものです。

- `README.md` の作成
- `sources/`、`notes/`、`outputs/`、`logs/` などの必要なフォルダ作成
- 初回のファイル配置
- 作業記録や次回メモの作成
- 動かすための設定ファイル作成

人間が最初に全部決める必要はありません。大事なのは、AIが作業できる場所を1つ渡すことです。

## 4. 最初から言わなくてよいこと

初心者が最初から考えすぎなくてよいものもあります。

- 完璧なフォルダ構成
- 長いプロンプト
- すべてのファイル名
- 細かいロードマップ
- 細かいルールや運用手順

最終的なゴールは伝えます。ただし、最初から細かい計画表まで作る必要はありません。

## 5. 作業後に確認する

作業が終わったら、最後にこれだけ聞きます。

<div className="copy-block">
<pre>{`今回作ったファイルとフォルダを一覧にしてください。
次に作業するとき、最初に読むべきファイルも教えてください。`}</pre>
</div>

これで次回からは、「続きやって」「前回の記録を見て進めて」で始めやすくなります。
