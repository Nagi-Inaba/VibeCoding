<p className="lead">最初からフォルダ構成を細かく作る必要はありません。まず作業フォルダを1つだけ作り、「ここで何をしたいか」をAIに伝えます。</p>

## 1. 作業フォルダを1つ作る

最初に人間がやるのは、作業場所を1つ決めることです。

```text
my-first-project/
```

この中に、最初から `sources/` や `outputs/` を手で作らなくても大丈夫です。  
必要なフォルダやファイルは、AIに作ってもらいます。

## 2. やりたいことを宣言する

作業フォルダを開いたら、最初の指示はこれくらいで十分です。

<div className="copy-block">
<pre>{`このフォルダを作業場所にします。
ここで、初心者向けの小さなガイドサイトを作りたいです。
必要なフォルダとファイルを作りながら、最初に動く形まで進めてください。
作業後に、作ったものと次にやることを短く教えてください。`}</pre>
</div>

アプリ、記事、資料、調査メモでも形は同じです。

<div className="copy-block">
<pre>{`このフォルダを作業場所にします。
ここで、〇〇を作りたいです。
必要なフォルダとファイルは、進めながら作ってください。
まず最小の形まで進めてください。`}</pre>
</div>

## 3. 細かい準備はAIに任せる

AIに任せてよいものは、たとえば次のようなものです。

- `README.md` の作成
- `sources/`、`notes/`、`outputs/`、`logs/` などの必要なフォルダ作成
- 初回のファイル配置
- 作業記録や次回メモの作成
- 動かすための設定ファイル作成

人間が最初に全部決める必要はありません。大事なのは、AIが作業できる場所を1つ渡すことです。

## 4. 最初から言わなくてよいこと

初心者が最初から考えすぎなくてよいものもあります。

- 完璧なフォルダ構成
- 長いプロンプト
- すべてのファイル名
- 将来の拡張計画
- 細かいルールや運用手順

必要になったら、その場でAIに作らせれば十分です。

## 5. 作業後に確認する

作業が終わったら、最後にこれだけ聞きます。

<div className="copy-block">
<pre>{`今回作ったファイルとフォルダを一覧にしてください。
次に作業するとき、最初に読むべきファイルも教えてください。`}</pre>
</div>

これで次回からは、「続きやって」「前回の記録を見て進めて」で始めやすくなります。
