# 初心者向け導入と日本語品質改善 設計

## 最終用途

AIコーディングツールをまだ使ったことがない読者が、契約関係を誤解せずに道具を1つ選び、作業フォルダを開くところまで迷わず進められるようにする。

## 現状の問題

- `はじめの一歩` が「作業フォルダを作る」から始まり、アカウント作成、プラン確認、ツールの導入が抜けている。
- ChatGPTを使う場合はCodex、Claudeを使う場合はClaude Codeという、実際に使う2つの始め方が説明されていない。
- `よく使うツール` は補助ツールから始まり、最初に必要なコーディング支援AIを扱っていない。
- 一部に「適当にダラダラ」「たぶん動いています」「かなり弱い」など、口語的または評価根拠が曖昧な表現が残る。
- `仕上げ・公開前` など、一般公開しない作業にも「公開」を前提とする見出しが残る。

## 設計方針

### 1. 最初に選ぶのは1つだけ

冒頭に、次の2ルートを並列で示す。

| すでに使っているもの・使いたい環境 | 選択肢 | 利用開始の考え方 |
| --- | --- | --- |
| ChatGPT Plus（月20ドル） | Codex | 新しいChatGPTデスクトップアプリでCodexを選ぶ。最初の一周にはPlusで十分とし、上限に達してから上位プランを検討する。 |
| Claude Pro（月20ドル） | Claude Code | Claude Proに含まれるClaude Codeを使う。最初の一周にはProで十分とし、上限に達してから上位プランを検討する。 |

複数契約を勧めず、「すでに使っているサービスがあれば、その系列から始める」「まだなければ公式の利用条件を比べて1つ選ぶ」と案内する。最初の一周は、ChatGPT PlusまたはClaude Proの月20ドルの個人向け基本有料プランで十分とする。価格には確認日、米国の月払い、税・地域差の注意を添える。

### 2. チャットAIとコーディング支援AIを分ける

ブラウザで会話するChatGPTやClaudeと、PC上の作業フォルダを読み書きできるCodex、Claude Codeを同じものとして扱わない。このサイトでは後者を使うことを説明する。

CodexとClaude CodeにはWeb、クラウド、CLI、IDEなど複数の使い方がある。このガイドの最初の一周では、PCの作業フォルダを直接開けるアプリ、CLI、IDE連携だけを対象にし、Webやクラウド上でリポジトリを操作する経路は分ける。OpenAI側は、2026年7月9日にChat、Work、Codexを統合した新しいChatGPTデスクトップアプリを正本の入口とする。

### 3. 導入完了を確認できる形にする

導入手順は次の順番にする。

1. コーディング支援AIを1つ選ぶ。
2. 公式ページで現在の利用条件を確認し、必要ならアカウント作成や有料プランの契約をする。
3. 公式手順でアプリ、CLI、IDE拡張のいずれかを導入してサインインする。
4. 空の作業フォルダを開く。
5. 読み取りだけの確認文を送り、AIがそのフォルダを認識できたことを確かめる。

### 4. 日本語は読者の行動と判断を主語にする

- 筆者の内輪の習慣に見える表現を一般化する。
- 相手を評価するような口語を避ける。
- 「安全」「必ず」「かなり」など根拠が必要な強い語を、確認可能な説明へ置き換える。
- 一般公開は条件付きの一経路とし、共有・継続利用・一般公開を混同しない。
- 製品名、外来語、略語は最初の使用箇所か用語集で説明する。

- `src/content/docs/start.mdx`: 導入の主ページ。AI選択、利用条件、導入確認を冒頭に追加する。
- `src/content/docs/tools.mdx`: 補助ツールの前に、最初に必要なコーディング支援AIへの案内を置く。口語表現を修正する。
- `src/content/docs/glossary.mdx`: コーディング支援AI、Codex、Claude Code、IDEを追加する。
- `src/pages/index.astro`: 最初の行動を「AIを1つ選ぶ」に合わせる。
- `src/content/docs/what-is-vibe-coding.mdx`: 「丸投げ」を具体的な責任分担の説明へ置き換える。
- `src/content/docs/asking.mdx`, `src/content/docs/templates.mdx`: 公開を前提に見える一般見出し・テンプレートを利用範囲ベースへ直す。
- `src/content/docs/short-prompts.mdx`: 高リスク判断ではAIだけで最終判断しないことを追記する。
- `src/content/docs/my-vibe-coding.mdx`: 内部語り、権威依存、曖昧な強調を読者向けに直す。
- `src/content/docs/common-mistakes.mdx`, `src/content/docs/recover.mdx`: 利用回数の節約を主目的に見せず、外部保存時の確認を具体化する。
- `src/content/docs/folder-structure.mdx`, `src/content/docs/publish-check.mdx`: 条件付き公開と利用範囲ベースの用語へ揃える。
- `astro.config.mjs`: 本文の見出し変更に合わせてサイドバーのラベルとアンカーを更新する。
- `scripts/content-contract.test.mjs`: 導入順、製品の区別、主要な表現ルールを固定する。

## 非対象

- 特定製品を「最もおすすめ」と決めること。
- 変動しやすい月額、利用回数、モデル名を本文に固定すること。
- OS別の長いインストール手順をサイト内へ複製すること。
- 一般公開の詳細手順を `はじめの一歩` に移すこと。

## 受け入れ条件

- `はじめの一歩` の最初の実作業が「コーディング支援AIを1つ選ぶ」になっている。
- ChatGPT/CodexとClaude/Claude Codeの2経路が区別されている。
- 2つを同時に契約する必要はなく、どちらか1つから始められると明記されている。
- 無料で試せる場合と有料契約が必要な場合があるため、公式ページで現在の条件を確認するよう案内している。
- ChatGPT PlusとClaude Proが米国月払い20ドルであり、このガイドの最初の一周には十分と案内している。
- 新しいChatGPTデスクトップアプリにChat、Work、Codexが統合されたことと、既存Codexアプリからの更新方法を案内している。
- 読者が導入後に送る、編集を伴わない確認文がある。
- 補助ツールが、最初に契約すべき必須ツールに見えない。
- Typelessの紹介報酬、クラウド処理、必要権限が明記されている。
- MarkItDown、Playwright、Product Starterの導入前提と利用境界が明記されている。
- GitHub検索の説明が、実際の `created:` と `--sort stars` の意味に一致している。
- 最初の依頼テンプレートがREADME 1ファイルに限定されている。
- 権利とアクセシビリティは、特定の人への共有と一般公開の両方で、成果物と利用者に応じて確認する。
- 法律、医療、お金に関わる判断は、AIだけで最終判断しないと案内している。
- 対象ページから、今回指定した口語・内部向け表現が除かれている。
- `npm run qa`、別のbase pathでのビルド、`git diff --check` が成功する。
